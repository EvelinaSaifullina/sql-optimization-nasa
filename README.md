# sql-optimization-nasa
SQL-оптимизация на реальных данных NASA (1.8 млн строк)

## 📌 Описание проекта

**Цель:** Доказать на практике, что правильно построенные индексы ускоряют SQL-запросы в десятки раз.

**Данные:** Логи HTTP-запросов к серверам NASA за июль 1995 (публичный датасет).  
- 1 886 838 строк исходных данных  
- 1 886 006 строк после очистки (удалено 832 дубликата)

## 🛠️ Этапы работы

### 1. Подготовка данных (SQL)
- Удаление дубликатов через `GROUP BY` + `MIN(id)`
- Нормализация URL (`LOWER(url)`) — объединено 290 вариантов
- Валидация IP-адресов (22.2% — IPv4, 77.8% — доменные имена)

### 2. Оптимизация запросов через индексы

| Эксперимент | Запрос | Без индекса | С индексом | Ускорение |
|-------------|--------|-------------|-------------|-----------|
| №1 | Поиск по IP `199.72.81.55` | 2611 мс | 117 мс | **22.3x** |
| №2 | Диапазон дат `BETWEEN` | 1642 мс | 177 мс | **9.3x** |
| №3 | Топ-50 IP с ошибкой 404 | 1607 мс | 136 мс | **11.8x** |

### 3. Ключевые выводы

- Индексы критически важны на объёме >1 млн строк
- Каждый индекс должен соответствовать паттерну запроса (универсальных нет)
- Подготовка данных — не опциональна: дубликаты, регистр, валидация
- В реальных данных 78% записей в поле `ip` — доменные имена, а не IPv4
